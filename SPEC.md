# ナップサック問題パズルゲーム「Knapsack Thief」仕様書

本作は、アルゴリズムの代表例である「ナップサック問題」をテーマにした、ブラウザで遊べる時間制限付きの選択型パズルゲームです。

---

## 1. 使用技術・システム環境
* **言語**: TypeScript (または JavaScript)
* **描画ライブラリ**: PixiJS (v8.x)
* **ビルドツール**: Vite
* **音響効果**: なし（音声アセット管理を不要とし、ミュートで動作）
* **グラフィックス**: すべて PixiJS Graphics API を用いたプログラム描画（外部画像ファイル不要）

---

## 2. 画面遷移と基本フロー
ゲームは以下の3つの画面で構成され、ステート管理により切り替わります。

1. **タイトル画面 (Title Screen)**
   * ゲームタイトル「Knapsack Thief」を表示。
   * 「ゲームスタート (Start Game)」ボタン。
2. **ゲーム画面 (Game Screen)**
   * 60秒の制限時間内に、棚にあるアイテムをバッグに詰め込み、価値（金額）を最大化するメイン画面。
   * 「脱出する（Complete）」ボタンでいつでも途中で切り上げて結果画面へ進むことが可能。
3. **結果画面 (Results Screen)**
   * 獲得したアイテムの総価値（スコア）と総重量を表示。
   * 盗み出したアイテムのリストを表示。
   * 獲得したスコアに応じた「ランク評価 (S〜D)」を表示。
   * 「タイトルに戻る (Back to Title)」ボタン。

---

## 3. ゲームのルールと操作仕様
* **目的**: 制限時間 60秒 以内に、ナップサック（バッグ）の**許容重量 50.0 kg** を超えないように、部屋に展示された宝物から合計価値（$）が最大になる組み合わせを選択する。
* **アイテムの選択**:
  * 棚に並んでいるアイテムをクリック（またはタップ）すると、バッグに追加（選択状態）される。
  * 選択されたアイテムは棚の上でハイライト（または半透明化・チェックマーク表示）され、バッグに入ったビジュアルとなる。
  * すでにバッグに入っているアイテムを再度クリックすると、選択が解除（バッグから取り出される）される。
* **重量制限の挙動 (重要)**:
  * アイテムをクリックした際、**「現在のバッグの総重量 ＋ 選択したアイテムの重量 ＞ 50.0 kg」** となる場合は、バッグに**追加できない**。
  * 追加できない場合は、画面上に一時的に「バッグの容量が足りません！」といった警告テキストやエフェクトを表示し、選択状態にはしない。
* **終了条件**:
  * 画面上の「脱出する」ボタンを押す。
  * または、制限時間（60秒）が 0秒 になる。

## 4. 画面レイアウト設計
画面解像度は `800x600` ピクセル（または可変・レスポンシブ）を想定。

### 4.1. タイトル画面
* 中央にゲームタイトル「Knapsack Thief（ナップサック・シーフ）」と簡単なルール説明を表示。
* 中央下部に「スタート」ボタンを配置。

### 4.2. ゲーム画面
左右に分割されたレイアウトを採用する。

* **左側エリア：宝物展示室 (Treasure Room)**
  * 木調の棚（グリッド）が描かれ、そこに15個の宝物が並んでいる。
  * 各宝物は PixiJS Graphics で描画された異なる形状・色のアイコンで表示される。
  * 宝物にマウスカーソルを**ホバー**すると、宝物の詳細（名前、重量、価値）がポップアップ（ツールチップ）または画面内の情報パネルに表示される。
* **右側エリア：バッグ＆ステータス (Bag & Status Panel)**
  * **タイマー表示**: 60秒からカウントダウン。
  * **バッグのビジュアル**: 大きな袋（ナップサック）のイラスト。
  * **重量ゲージ**: 現在の総重量 / 50.0 kg を視覚的に表すプログレスバー。
  * **合計価値**: 現在バッグに入っているアイテムの合計金額（例: `$12,500`）を大きく表示。
  * **脱出ボタン**: 赤色などの目立つボタンで「脱出する（ゲーム完了）」を配置。

### 4.3. 結果画面
* リザルトの統計情報を中央に表示。
* **スコア表示**: 「盗み出したお宝の総額: `$XXXXX`」
* **重量表示**: 「総重量: `XX.X kg / 50.0 kg`」
* **評価ランク**:
  * プレイヤーのスコアに応じて、以下のランクを判定・表示する。
    * **Sランク** (超一流の怪盗): `$15,000` 以上（最適解 `$15,800` に非常に近い）
    * **Aランク** (一流の泥棒): `$14,000` 〜 `$14,999`
    * **Bランク** (中堅の空き巣): `$12,000` 〜 `$13,999`
    * **Cランク** (駆け出しの泥棒): `$10,000` 〜 `$11,999`
    * **Dランク** (おっちょこちょい): `$10,000` 未満
* 獲得したアイテムのミニアイコンまたは名前リスト。
* 「タイトルに戻る」ボタン。

## 6. 実装のベストプラクティス
1. **ステート管理**:
   * 現在のゲーム状態（`TITLE`, `PLAYING`, `RESULTS`）を定義し、状態に応じてコンテナの表示・非表示を切り替える。
2. **ホバー演出とツールチップ**:
   * `PointerOver` イベントで宝物のスケールを少し大きくし、白枠などのアウトラインを付与する。
   * マウス座標、あるいは画面固定の情報表示領域にそのアイテムのパラメータを表示する。
3. **容量制限時のフィードバック**:
   * 重量オーバーのアイテムをクリックした際、バッグの容量テキストやゲージを赤く点滅させ、警告音の代わりに「容量オーバー！」と画面中央付近にフェードアウトするテキストを表示する。
4. **レスポンシブスケール**:
   * PixiJS の Application の `resize` イベント、または CSS の flexbox 等を用いて、アスペクト比を維持しながら画面中央にフィットするように配置する。
